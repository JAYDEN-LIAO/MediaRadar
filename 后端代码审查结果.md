## 一、总体评估

### 健康度评分：**5.5 / 10**

**优点：**
- 整体架构清晰：FastAPI 网关 + 多服务模块化 + SQLite 单库共享（除 agent_memory 独立 DB）
- 认证链路完整：登录 → JWT 签发 → `get_current_user` 依赖 → tenant 隔离
- 配置管理集中：`core/config.py` 统一读取 `.env`
- 推送抽象层（Notifier 包）设计合理，4 个 channel 一致性良好
- LangGraph 子图（analyst → reviewer → director）实现完整
- 26 工具 + RSS 通道 + 4 类型订阅均已落地

**严重问题：**
- 多租户隔离在多处**几乎完全缺失**（subscription 全局聚合、memory 表无 owner_id 字段）
- **CLAUDE.md 与实际 API 严重脱节**（文档 12 端点，实际 36+）
- crawler_service 独立运行在 8080 端口，**完全绕过 gateway 的鉴权 / CORS / 限流**
- 关键 SQL 字段索引错误（`tools/system.py:341`）
- 死代码较多（10+ 处冗余 import、3 个完整废弃函数、1 个未触发熔断器）
